# LOL 内战分队系统

基于 WebRTC 的纯前端实现，无需服务器，浏览器之间直接通信。

---

## 解决了什么问题？

开黑玩 LOL 自定义游戏（内战）时，普通的随机分队经常遇到这些问题：

1. **实力不均** —— 随机分队可能导致两边实力差距过大，游戏一边倒，体验差
2. **有人不服** —— 随机结果不满意时，总有人质疑分队的公平性
3. **私人恩怨** —— "我不想跟他一队"、"他俩不能在同一队"，随机分队无法照顾这些需求

**这个工具通过「投票选队长 + 偏好收集 + 智能算法」来解决这些问题**，让分队结果更合理，大家更愿意接受。

## 使用流程

### 1. 创建/加入房间

- **房主**：点击"创建房间"，输入昵称，获得 6 位房间号
- **其他玩家**：点击"加入房间"，输入房间号和昵称

### 2. 第一阶段：投票选队长

- 每人可投 0-2 票，选择想让谁当队长
- 所有人都确认后自动选出得票最高的 2 人
- 房主可提前结束投票

### 3. 第二阶段：提交偏好

**普通玩家：**
- 选择倾向的队伍（跟随某队长）
- 选择最多 1 名不想同队的玩家

**队长：**
- 选择心仪的队员（可多选）

### 4. 第三阶段：分队结果

- 系统根据权重算法自动分队
- 显示最终队伍分配
- 房主可查看算法日志、重新打乱或彻底重来

## 注意事项

- **房主关闭页面后，房间数据丢失**（P2P 特性）
- 所有玩家需要能访问 PeerJS 信令服务器
- 分队算法仅在房主端执行，确保结果一致
- 建议 10 人使用（5v5），最少需要 2 人

---

## 算法说明

分队算法基于加权概率系统：

- **基础权重**：两队均为 1.0
- **个人偏好**：+2.0
- **队长心仪**：+2.5
- **规避惩罚**：乘以规避系数（0.0-1.0，由房主设置）

最终概率：`P(A) = Weight_A / (Weight_A + Weight_B)`
